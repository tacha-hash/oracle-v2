# Parallel Agent Testing Pattern

**Date**: 2025-12-24
**Context**: Testing Oracle v2 commands with multiple agents

## What We Learned

- **Parallel Task calls** - Can launch 4+ agents simultaneously in one message
- **Haiku model** - Fast, cheap, perfect for context-finder tasks
- **context-finder agent** - Handles search/gather across git, files, issues
- **Consistent output format** - Markdown tables work well across all commands

## Why It Matters

- Speed: 4 commands tested in ~5 seconds total vs ~20 seconds sequential
- Cost: Haiku is much cheaper than Opus for simple tasks
- Pattern: Agent specialization (context-finder for search, executor for actions)

## How To Apply

```
// Launch multiple agents in parallel - single message, multiple Task tool calls
Task 1: subagent_type=context-finder, model=haiku, prompt="..."
Task 2: subagent_type=context-finder, model=haiku, prompt="..."
Task 3: subagent_type=context-finder, model=haiku, prompt="..."
Task 4: subagent_type=context-finder, model=haiku, prompt="..."
// All execute simultaneously, results return together
```

## Agent Types Available

| Agent | Model | Purpose |
|-------|-------|---------|
| context-finder | haiku | Fast search (git, files, issues) |
| executor | haiku | Execute plans from issues |
| marie-kondo | haiku | File placement consultant |
